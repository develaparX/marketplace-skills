# Get Creator Profile API Implementation Guide

## 1. Purpose & Use Case
API gets creator profile info. Use when verifying creator region, permissions, or open ID.

## 2. Endpoint & Method
*   **Method**: `GET`
*   **Path**: `/affiliate_creator/202508/profiles`
*   **Base URL**: `https://open-api.tiktokglobalshop.com`

## 3. Headers & Auth
*   `content-type`: `application/json`
*   `x-tts-access-token`: Creator access token (from Get Access Token, where `user_type` = 1).

## 4. Query Parameters
*   `app_key` (string, required): Unique app key.
*   `sign` (string, required): Signature generated by algorithm.
*   `timestamp` (number, required): Unix timestamp GMT (UTC+00:00).

## 5. Response Structure
*   `code` (number): Status code. `0` means success.
*   `message` (string): Status message. Contains failure reasons.
*   `request_id` (string): Request log identifier.
*   `data` (object): Profile details.
    *   `avatar` (object): Creator avatar image.
        *   `width` (number): Image width.
        *   `height` (number): Image height.
        *   `url` (string): Image URL.
    *   `username` (string): Creator username.
    *   `selection_region` (string): Selection region (e.g., "US").
    *   `register_region` (string): Register region (e.g., "US").
    *   `seller_type` (string): Seller type (e.g., "LOCAL").
    *   `permissions` (array of strings): Creator permissions list.
    *   `user_type` (string): Account type.
    *   `creator_user_open_id` (string): Creator open ID.

## 6. Error Handling
Handle these specific error codes:
*   `16015006`: No selection region. Creator must enable EC permission.
*   `16015007`: No sale region. Creator lacks sale region.
*   `16501011`: No permission. Check creator auth info.
*   `16504002`: Query failed. Verify creator EC permission is active.
*   `36009002`: Rate limit hit. Implement backoff.

## 7. Pitfalls & Best Practices
*   **Token Type**: Ensure access token comes from `user_type` = 1 flow. Other types fail auth.
*   **Signature Order**: Generate `sign` using exact query parameter order. Wrong order causes signature mismatch.
*   **EC Permission**: Check `permissions` array for `ADD_AFFILIATE_PERMISSION` before running affiliate tasks.
