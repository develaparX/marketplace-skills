# FBT Update Goods API

## 5. Response Structure
*   `code` (number): `0` for success. Non-zero for error.
*   `message` (string): Status description.
*   `request_id` (string): Debugging log ID.
*   `data` (object): Contains update results.
    *   `update_result_info` (object):
        *   `tts_goods_id` (string): Target goods ID.
        *   `is_success` (boolean): Update status.
        *   `fail_reason` (string): Error details if `is_success` is false.

## 6. Error Handling
*   Check `code` first. If `code` != `0`, API request failed.
*   Check `data.update_result_info.is_success`. If `false`, check `fail_reason`.
*   Log `request_id` for support tickets.

## 7. Pitfalls & Best Practices
*   **Signature:** Generate `sign` using sorted query parameters. Wrong order causes auth failure.
*   **Timestamp:** Must be current UTC. Server rejects requests older than threshold.
*   **Image URL:** Must be publicly accessible. Escape ampersands (`&`) in URL query strings.
