# API Implementation Guide: Get Opportunity Detail

## 1. Overview
Retrieve full details of single opportunity lead. Use to get market trends, competition level, recommended actions.

## 2. Endpoint
* **Method:** `GET`
* **Path:** `/product/202604/opportunities/{opportunity_id}`

## 3. Headers & Auth
* `Content-Type`: `application/json`
* `x-tts-access-token`: Seller access token (from Get Access Token, user_type = 0).

## 4. Parameters

| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `opportunity_id` | string | Yes | Unique ID of opportunity lead. |

### Query Parameters
| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique key assigned to app. |
| `sign` | string | Yes | Signature generated by algorithm. |
| `timestamp` | number | Yes | Unix timestamp GMT (UTC+00:00). |
| `shop_cipher` | string | Yes | Shop identifier. |
| `locale` | string | No | BCP-47 format (e.g., `en-US`). |

## 5. Response Structure
```json
{
  "code": 0,
  "message": "Success",
  "request_id": "20260401...",
  "data": {}
}
```
* `code` (number): Status code. `0` means success.
* `message` (string): Status message.
* `request_id` (string): Log identifier for debugging.
* `data` (object): Opportunity details.

## 6. Error Handling
* Check `code` field. Non-zero value indicates failure.
* Log `request_id` for support tickets.

## 7. Pitfalls & Best Practices
* **Timestamp drift:** Sync system clock with UTC. Signature fails if timestamp differs from server time.
* **Signature generation:** Include all query parameters in signature calculation. Sort keys alphabetically.
* **Access token:** Token must belong to seller (`user_type = 0`).
