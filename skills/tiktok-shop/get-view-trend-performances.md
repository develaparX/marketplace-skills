## 6. Error Handling
Handle specific error codes:
*   `66009302`: Invalid parameter. Check `live_room_id` format.
*   `66009315`: No permission. Check token validity and scopes.

## 7. Pitfalls & Best Practices
*   **Token Type:** API fails if token is seller type. Use creator token (`user_type = 1`).
*   **Signature:** Generate `sign` using sorted query parameters. Incorrect order causes auth failure.
*   **Time Sync:** Server clock must sync with UTC. Out-of-sync timestamp rejects request.
