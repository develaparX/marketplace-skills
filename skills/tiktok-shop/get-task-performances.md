# Get Task Performances API Implementation Guide

## 1. Overview
API retrieves performance metrics. Use to track reads, orders, GMV, coupons for engagement tasks.

## 2. Endpoint
*   **Method:** `POST`
*   **Base URL:** `https://open-api.tiktokglobalshop.com`
*   **Path:** `/customer_engagement/202412/performances`

## 3. Headers & Authentication
*   `content-type`: `application/json`
*   `x-tts-access-token`: Seller access token (from Get Access Token, `user_type` must be `0`).

## 4. Parameters

### Query Parameters
| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique app key. |
| `sign` | string | Yes | Signature generated by algorithm. |
| `timestamp` | number | Yes | Unix timestamp GMT (UTC+00:00). |
| `shop_cipher` | string | Yes | Shop cipher for target shop. |

### Body Parameters (JSON)
| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `task_ids` | array | Yes | List of task IDs to query. |

## 5. Response Structure
Response returns JSON object:

*   `code` (number): Status code. `0` means success.
*   `message` (string): Status description.
*   `request_id` (string): Log identifier.
*   `data` (object): Payload container.
    *   `task_performances` (array): Performance list.
        *   `id` (string): Task ID.
        *   `status` (string): Task status (e.g., `SENDING`).
        *   `sent_recipient_count` (number): Sent count.
        *   `read_recipient_count` (number): Read count.
        *   `order_count` (number): Converted orders.
        *   `gmv_amount` (string): GMV generated.
        *   `claimed_coupons_count` (number): Claimed coupons.

## 6. Error Handling
Check `code` field. Code `0` is success. Non-zero is error. Use `request_id` to debug with support.

## 7. Pitfalls & Best Practices
*   `gmv_amount` returns as string. Parse to float before math operations.
*   Do not mix query and body params. Put `task_ids` in JSON body.
*   Keep system clock synced. Old timestamp causes signature error.
