# Implement Search Global Products API

## 1. Purpose & Use Case
Search global products. Filter by status, SKU, creation time, update time. Use for inventory sync, product audits.

## 2. Endpoint & Method
* **Method:** `POST`
* **Path:** `/product/202312/global_products/search`

## 3. Headers & Auth
* **Scope required:** `seller.global_product.info`
* **Headers:**
  * `content-type`: `application/json` (Required)
  * `x-tts-access-token`: Seller access token (Required)

## 4. Parameters

### Query Parameters
| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique app key |
| `sign` | string | Yes | Signature generated by gen algorithm |
| `timestamp` | int | Yes | Unix timestamp GMT (UTC+00:00) |
| `page_size` | int | Yes | Results per page. Range: [1-100] |
| `page_token` | string | No | Token for next page |

### Request Body (JSON)
| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `status` | string | No | Filter: `PUBLISHED`, `UNPUBLISHED`, `DRAFT`, `DELETED` |
| `seller_skus` | array of strings | No | Filter by seller SKU codes |
| `create_time_ge` | int | No | Created on or after. Unix timestamp |
| `create_time_le` | int | No | Created on or before. Unix timestamp |
| `update_time_ge` | int | No | Updated on or after. Unix timestamp |
| `update_time_le` | int | No | Updated on or before. Unix timestamp |

## 5. Response Structure
JSON object:
* `code` (int): Status code. `0` usually means success.
* `message` (string): Error or success message.
* `request_id` (string): Request log ID.
* `data` (object): Return info. Contains `next_page_token` for pagination.

## 6. Error Handling
* Check `code` in response. Non-zero indicates error.
* Log `request_id` to debug failed requests.
* Retry on 5xx errors. Do not retry on 4xx errors.

## 7. Pitfalls & Best Practices
* **Page Size:** Keep `page_size` between 1 and 100. Out of range causes error.
* **Pagination:** Use `next_page_token` from response `data` in next request `page_token` query parameter.
* **Signatures:** Generate `sign` using correct algorithm. Ensure `timestamp` matches generation time.
* **Timestamps:** Use Unix epoch seconds for time filters.
