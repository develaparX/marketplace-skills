# API Implementation Guide: Check Product Listing

## 1. Purpose & Use Case
Validate product listing data before submission. Use before create or update product calls to catch validation errors early.

## 2. Endpoint & Method
*   **Method:** `POST`
*   **Path:** `/product/202309/products/listing_check`

## 3. Headers & Authentication
### Required Scopes
*   `seller.product.basic`
*   `seller.product.optimize`

### Headers
| Header | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `content-type` | string | Yes | Must be `application/json` |
| `x-tts-access-token` | string | Yes | Seller access token |

## 4. Parameters

### Query Parameters
| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique app key |
| `sign` | string | Yes | Signature generated by algorithm |
| `timestamp` | int | Yes | Unix timestamp GMT (UTC+00:00) |
| `shop_cipher` | string | Yes | Shop information cipher |
| `is_diagnosis_required` | bool | No | **Deprecated.** Flag for listing quality info |

### Body Parameters (JSON)
| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `description` | string | Yes | Product description. HTML format. Max 10,000 characters. |
| `category_id` | string | Yes | Leaf category ID. |
| `brand_id` | string | No | Brand ID. |

## 5. Response Structure
*Note: API schema defines no specific response fields. Standard response wrapper expected:*

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

## 6. Error Handling
*   **400 Bad Request:** Invalid HTML, description too long, or category ID not leaf node.
*   **401 Unauthorized:** Missing or expired `x-tts-access-token`.
*   **403 Forbidden:** Missing required scopes.
*   **Action:** Parse response JSON `code` and `message` fields for validation failure details.

## 7. Pitfalls & Best Practices
*   **Description Length:** Limit HTML string to 10,000 characters. Count includes HTML tags.
*   **Category ID:** Must use leaf category ID. Parent category IDs cause validation failure.
*   **Deprecated Field:** Do not use `is_diagnosis_required`.
*   **Signature:** Generate `sign` query param using standard algorithm before call.
