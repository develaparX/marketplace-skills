# Get Interactive Trend Performances API Implementation Guide

## 1. Overview
API fetch live room interactive performance trend data. Use to track viewer engagement metrics over time.

## 2. Endpoint
* **Method**: `GET`
* **URL**: `https://open-api.tiktokglobalshop.com/analytics/202502/live_rooms/{live_room_id}/interactive_trend_performances`

## 3. Headers & Auth
* `content-type`: `application/json`
* `x-tts-access-token`: Creator access token (from Get Access Token, user_type = 1).

## 4. Parameters
### Path Parameter
* `live_room_id` (string, required): Live stream room ID.

### Query Parameters
* `app_key` (string, required): Unique app key.
* `sign` (string, required): Signature generated by algorithm.
* `timestamp` (number, required): Unix timestamp GMT (UTC+00:00).

## 5. Response Structure
JSON response contains:
* `code` (number): Status code. `0` means success.
* `message` (string): Status message.
* `request_id` (string): Request log ID.
* `data` (object): Return payload.
  * `interactive_trend_performances` (array): Trend data.
    * `stats_type` (string): Metric type (e.g., `WATCH_PV`).
    * `data_points` (array): Data points.
      * `value` (string): Metric value.
      * `timestamp` (number): Data point timestamp.

## 6. Error Handling
Check `code` field.
* `66009302`: Invalid room ID. Check `live_room_id` path parameter.
* `66009315`: No permission. Check access token scope.

## 7. Pitfalls & Best Practices
* **Timestamp**: Use UTC Unix timestamp. Request fail if drift too large.
* **Signature**: Generate `sign` using exact query parameters and app secret.
* **Token**: Ensure token belongs to creator (`user_type = 1`).
