# Get Traffic Performances API Implementation Guide

## 1. Overview
Get live stream traffic data. Use for viewer source analysis.

## 2. Endpoint
*   **Method**: `GET`
*   **Base URL**: `https://open-api.tiktokglobalshop.com`
*   **Path**: `/analytics/202502/live_rooms/{live_room_id}/traffic_performances`

## 3. Headers & Authentication
*   `content-type`: `application/json`
*   `x-tts-access-token`: Creator access token (`user_type = 1`).

## 4. Parameters

### Path Parameters
*   `live_room_id` (string, required): Live stream room ID.

### Query Parameters
*   `app_key` (string, required): Unique app key.
*   `sign` (string, required): Signature generated by algorithm.
*   `timestamp` (number, required): Unix timestamp GMT (UTC+00:00).

## 5. Response Structure
JSON object containing:
*   `code` (number): Status code. `0` is success.
*   `message` (string): Error details or success message.
*   `request_id` (string): Log identifier.
*   `data` (object): Return payload.
    *   `traffic_performances` (array): Traffic source list.
        *   `source` (object): Main source.
            *   `name` (string): Source name (e.g., `card_click`).
            *   `watch_pv` (number): Page views.
        *   `sub_sources` (array): Sub-source list. Same structure as `source`.

## 6. Error Handling
Check `code` in response.

| Code | Description | Action |
| :--- | :--- | :--- |
| `66009302` | Invalid param. | Check `live_room_id` format. |
| `66009315` | No permission. | Verify token scope and `user_type = 1`. |

## 7. Pitfalls & Best Practices
*   **Token Type**: Must use creator token. Merchant token causes auth failure.
*   **Signature**: Generate `sign` using correct algorithm. Timestamp must be current UTC.
