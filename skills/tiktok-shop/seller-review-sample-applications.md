# Seller Review Sample Applications API Implementation Guide

## 1. Purpose & Use Case
Seller approve or reject creator sample application in open collaboration. Use when creator request sample product and seller must decide.

## 2. Endpoint & Method
*   **Method:** `POST`
*   **Base URL:** `https://open-api.tiktokglobalshop.com`
*   **Path:** `/affiliate_seller/202507/sample_applications/{application_id}/review`

## 3. Headers & Auth
*   `content-type`: `application/json`
*   `x-tts-access-token`: Seller access token (from Get Access Token, `user_type = 0`).

## 4. Parameters

### Path Parameter
*   `application_id` (string, required): Sample request application ID.

### Query Parameters
*   `app_key` (string, required): Unique app key.
*   `sign` (string, required): Signature generated by algorithm.
*   `timestamp` (number, required): Unix timestamp GMT (UTC+00:00).
*   `shop_cipher` (string, required): Shop identifier.

### Request Body (JSON)
*   `review_result` (string, required): Audit action. Values: `APPROVE`, `REJECT`.
*   `reject_reason` (string, conditional): Required if `review_result` is `REJECT`. Values: `NOT_MATCH`, `OFFLINE`, `OUT_OF_STOCK`, `OTHER`.

## 5. Response Structure
*   `code` (number): Status code (0 is success).
*   `message` (string): Status message.
*   `request_id` (string): Request log identifier.
*   `data` (object): Specific return information.

## 6. Error Handling
Check `code` in response.
*   `16022020`: Seller has no permission to operate agency sample request. Check shop ownership.

## 7. Pitfalls & Best Practices
*   Include `reject_reason` when `review_result` is `REJECT`. API fail if missing.
*   Sync system clock. Outdated `timestamp` cause signature validation failure.
*   Verify token type. Must use seller token (`user_type = 0`).
