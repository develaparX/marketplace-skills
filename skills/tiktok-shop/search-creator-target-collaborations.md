## 5. Response Structure
Response return status, data object.

* `code` (number): Status code. `0` is success.
* `message` (string): Status message.
* `request_id` (string): Request log ID.
* `data` (object):
  * `total_count` (number): Total results.
  * `next_page_token` (string): Token for next page.
  * `target_collaborations` (array):
    * `id` (string): Collaboration ID.
    * `name` (string): Collaboration name.
    * `status` (string): Collaboration status (e.g., `LIVE`).
    * `products` (array):
      * `id` (string): Product ID.
      * `title` (string): Product title.
      * `main_image_url` (string): Image URL.
      * `commission` (object):
        * `rate` (number): Commission rate (e.g., 1000 = 10%).
        * `amount` (string): Commission amount.
        * `currency` (string): Currency code (e.g., `USD`).

## 6. Error Handling
Check `code` field. `0` is success. Non-zero is error. Save `request_id` for support logs.

## 7. Pitfalls & Best Practices
* **Token Type**: Use creator token (`user_type = 1`). Shop token fail.
* **Page Size**: Keep `page_size` under 100. Large request fail.
* **Sign Check**: Generate `sign` using current timestamp. Old timestamp fail.
