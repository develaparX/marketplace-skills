# TikTok Shop API: Create First Mile Bundle

## 1. Purpose
Combine multiple packages into single first-mile bundle for TikTok Shop warehouse. Use when shipping multiple orders together. Returns bundle ID.

## 2. Endpoint
*   **Method:** `POST`
*   **Path:** `/fulfillment/202407/bundles`
*   **Base URL:** `https://open-api.tiktokglobalshop.com`

## 3. Headers & Auth
### Headers
*   `content-type`: `application/json`
*   `x-tts-access-token`: Seller access token (from Get Access Token, user_type = 0)

### Query Parameters (Auth)
*   `app_key`: Unique app key.
*   `sign`: Signature generated by algorithm.
*   `timestamp`: Unix timestamp GMT (UTC+00:00).

## 4. Parameters
### Request Body (JSON)
| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `order_ids` | array[string] | Yes | IDs of orders in bundle. |
| `handover_method` | string | Yes | Shipping method. Values: `PICKUP`, `DROP_OFF`. |
| `shipping_provider_id` | string | Conditional | Logistics provider ID. Required if `handover_method` is `DROP_OFF`. |
| `tracking_number` | string | Conditional | Bundle tracking number. Required if `handover_method` is `DROP_OFF`. |
| `phone_tail_number` | string | Conditional | Last 4 digits of sender phone. Required if `handover_method` is `DROP_OFF`. |

## 5. Response Structure
### JSON Fields
*   `code` (number): Status code. `0` means success.
*   `message` (string): Status message.
*   `request_id` (string): Request log ID.
*   `data` (object): Bundle details.
    *   `first_mile_bundle_id` (string): Created bundle ID.
    *   `url` (string): Document URL.
    *   `errors` (array): Failed packages list.
        *   `code` (number): Error code.
        *   `message` (string): Error description.
        *   `detail` (object): Error details (e.g., `order_id`).

## 6. Error Handling
*   Check root `code`. If not `0`, API call failed.
*   Check `data.errors` array. API can return success code `0` but contain failed orders inside `data.errors`.
*   Example error code: `10007014` (invalid package id).

## 7. Pitfalls & Best Practices
*   **Conditional Fields:** Do not omit `shipping_provider_id`, `tracking_number`, or `phone_tail_number` if `handover_method` is `DROP_OFF`. API will fail.
*   **Partial Success:** Always parse `data.errors` even if HTTP status is 200 and `code` is `0`. Some orders in bundle may fail while others succeed.
*   **Signature:** Generate `sign` using exact query parameters and request body. Mismatched timestamp or app_key causes auth failure.
