# Implementation Guide: Get Bestselling Creators API

## 1. Purpose
API fetches top 100 performing creators. Use to identify high-value affiliate partners.

## 2. Endpoint
*   **Method:** `GET`
*   **URL:** `https://open-api.tiktokglobalshop.com/analytics/202511/creators/bestselling`

## 3. Headers & Authentication
*   `content-type`: `application/json`
*   `x-tts-access-token`: Seller access token. Required. Obtain via Get Access Token API (user_type must be 0).

## 4. Query Parameters

| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique application key. |
| `sign` | string | Yes | Signature generated by algorithm. |
| `timestamp` | number | Yes | Unix timestamp GMT (UTC+00:00). |
| `date` | string | Yes | Reference date. Format: `YYYY-MM-DD`. Use shop registered timezone. |
| `time_slot` | string | Yes | Query time range. Allowed: `1D`, `7D`, `30D`. |
| `author_type` | string | Yes | Creator type. Allowed: `OFFICIAL`, `CHANNEL`, `AFFILIATE`. |
| `shop_cipher` | string | Yes | Shop identifier cipher. |
| `currency` | string | No | Currency filter. Allowed: `USD`, `LOCAL`. |

## 6. Error Handling
*   Check `code` field. `0` indicates success.
*   Non-zero `code` indicates failure. Log `request_id` and `message` for debugging.

## 7. Pitfalls & Best Practices
*   **Timezone Alignment:** Use shop registered timezone for `date` parameter. Wrong timezone causes empty data.
*   **Signature Generation:** Sort all query parameters alphabetically before hashing. Include all required parameters in signature calculation.
*   **Caching:** Cache response data. Creator rankings update daily. Avoid redundant API calls to prevent rate limiting.
