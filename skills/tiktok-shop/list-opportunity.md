## 6. Error Handling
*   Check `code` field. Non-zero value means error.
*   Save `request_id` in logs. Use to debug with support.

## 7. Pitfalls & Best Practices
*   `page_size` limit: Do not exceed 100. Request fails if too large.
*   `opportunity_type` value: Must use uppercase `PRODUCT`, `KEYWORD`, or `CATEGORY`.
*   `timestamp` check: Must use UTC time. Server rejects expired requests.
