# API Implementation: Update Stock Operation Settings

Configure auto-replenishment for SKUs. Use when automating inventory stock levels in bulk.

*   **Method:** `POST`
*   **Path:** `/product/202604/inventory/operation/settings`

## Headers & Authentication

*   `Content-Type`: `application/json`
*   `x-tts-access-token`: Seller access token.

| Name | Type | Required | Description |

| `seller_id` | string | Yes | Identifies seller whose settings change. |

| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `sku_ids` | array | No | List of SKU IDs. Max 500 per batch. |
| `stock_operation_setting` | object | No | Stock operation configuration. |

## Response Structure

```json
{
  "code": 0,
  "message": "success",
  "request_id": "20260401...",
  "data": {}
}
```

*   `code` (number): Status code. `0` means success.
*   `message` (string): Status description.
*   `request_id` (string): Log identifier for debugging.
*   `data` (object): Return payload.

## Error Handling

1. Check `code` field in response. Non-zero indicates failure.
2. Log `request_id` for API support troubleshooting.
3. Retry request on network timeout. Do not retry on authorization errors.

## Pitfalls & Best Practices

*   **Batch Limit:** Do not exceed 500 SKUs in `sku_ids`. Split larger lists into multiple requests.
*   **Signature Expiry:** Generate `sign` fresh for each request. Old timestamps cause authentication failure.
