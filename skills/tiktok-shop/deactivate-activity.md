# Deactivate Activity API

## Purpose
Stop ongoing or upcoming promotion activity. Use when promotion must end early.

## Endpoint
* **Method**: `POST`
* **Path**: `/promotion/202309/activities/{activity_id}/deactivate`

## Headers
* `content-type`: `application/json`
* `x-tts-access-token`: Seller access token. Get from OAuth flow (user_type = 0).

## Parameters

### Path Parameters
* `activity_id` (string, required): Target activity ID.

### Query Parameters
* `app_key` (string, required): App unique key.
* `sign` (string, required): Request signature generated by algorithm.
* `timestamp` (number, required): Unix timestamp (UTC).
* `shop_cipher` (string, required): Shop identifier. Get via Get Authorization Shop API.

## Response

### Fields
* `code` (number): Status code. 0 means success.
* `message` (string): Status message. Contains failure reason.
* `request_id` (string): Request log ID.
* `data` (object): Return payload.
  * `activity_id` (string): Deactivated activity ID.
  * `title` (string): Activity title.
  * `status` (string): Activity status.
  * `update_time` (number): Update timestamp.

## Error Codes
* `17029001`: Invalid parameters
* `17029009`: Activity does not exist
* `17029010`: Activity is deactivated
* `17029011`: Not allowed to update beginning time of activity
* `17029012`: Not allowed to update expired activities
* `17029028`: You are specifying activities not in your shop
* `17029036`: ActivityType is not supported
* `17029052`: ProductType is not supported
* `17029055`: Request Timeout
* `36009003`: Internal error. Try again.

## Pitfalls & Best Practices
* **Shop Cipher**: Cross-border shops require correct `shop_cipher`. Wrong cipher causes failure.
* **Expired Activities**: Cannot deactivate expired activities. Check status before request.
* **Signature**: Generate signature using exact path and query parameters.
* **Request Body**: Send empty JSON object `{}` in body.
