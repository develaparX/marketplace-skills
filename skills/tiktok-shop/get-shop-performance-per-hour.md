# Shop Performance Per Hour API

## 1. Overview
API gets hourly shop performance metrics. Use for daily dashboard, hourly sales tracking. 
* **Limit**: Data available for past 30 days (includes today).

## 2. Endpoint
* **Method**: `GET`
* **URL**: `https://open-api.tiktokglobalshop.com/analytics/202510/shop/performance/{date}/performance_per_hour`

## 3. Headers & Auth
* `content-type`: `application/json`
* `x-tts-access-token`: Seller access token (from Get Access Token, user_type = 0).

## 4. Parameters

### Path Parameters
| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `date` | string | Yes | ISO 8601 format (`YYYY-MM-DD`). Use shop registered timezone. |

### Query Parameters
| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique app key. |
| `sign` | string | Yes | Signature generated by algorithm. |
| `timestamp` | number | Yes | Unix timestamp GMT (UTC+00:00) in seconds. |
| `shop_cipher` | string | Yes | Shop cipher identifier. |
| `currency` | string | No | Values: `USD`, `LOCAL`. Default: `LOCAL`. |

## 5. Response Structure
Returns JSON object.

* `code` (number): Status code. `0` means success.
* `message` (string): Status message.
* `request_id` (string): Log identifier.
* `data` (object):
  * `performance` (object):
    * `overall` (object): Total day metrics.
      * `gmv` (object): `amount` (string), `currency` (string).
      * `items_sold` (number): Total items sold.
      * `visitors` (number): Total unique visitors.
      * `customers` (number): Total unique buyers.
    * `intervals` (array): Hourly breakdown.
      * `index` (number): Hour of day (0-23).
      * `gmv` (object): Hourly GMV.
      * `items_sold` (number): Hourly items sold.
      * `visitors` (number): Hourly unique visitors.

## 6. Error Handling
* Check `code` field. If `code` not `0`, request failed.
* Log `request_id` for debugging.

## 7. Pitfalls & Best Practices
* **Timezone**: Query using shop timezone. Wrong timezone shifts hourly data.
* **Retention**: 30-day limit. Store data locally for long-term analysis.
* **Signature**: Generate `sign` query param last. Include all query parameters in signature calculation.
