# Send Engagement Message API Implementation Guide

## 1. Overview
Send messages to buyers for engagement tasks. Use when target buyers placed order in past 365 days.

* `x-tts-access-token`: Seller access token. Required. Obtain via Get Access Token (`user_type = 0`).

## 5. Response Structure
* `code` (number): Status code. `0` means request processed.
* `message` (string): Status description.
* `request_id` (string): Log identifier for debugging.
* `data` (object): Contains execution details, including partial failures.

## 6. Error Handling
Handle specific error codes in application logic:

| Code | Description | Action |
| :--- | :--- | :--- |
| `68009001` | Shop lacks CRM access. | Stop request. Verify seller meets CRM criteria. |
| `68009002` | Task ended. | Stop request. Check task status. |
| `68009003` | Invalid buyer email. | Remove email from list. |
| `68009004` | No order in past 365 days. | Filter buyer out. Target active buyers only. |

## 7. Pitfalls & Best Practices
* **Partial Success**: API returns `code: 0` even if individual emails fail. Check `data.errors` array in response to identify failed emails.
* **365-Day Limit**: Verify buyer order history before sending. Prevents error `68009004`.
* **Token Type**: Use seller token (`user_type = 0`). Partner tokens fail.
