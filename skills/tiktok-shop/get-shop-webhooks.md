# Get Shop Webhooks Implementation Guide

## 1. Overview
Retrieve active webhooks and target URLs for shop. Use to verify webhook setup or debug event delivery.

## 2. Endpoint
*   **Method:** `GET`
*   **URL:** `https://open-api.tiktokglobalshop.com/event/202309/webhooks`

## 3. Headers & Authentication
*   `content-type`: `application/json`
*   `x-tts-access-token`: Seller access token (from OAuth, `user_type` = 0).

## 4. Query Parameters

| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique developer application key. |
| `timestamp` | number | Yes | Unix timestamp GMT (UTC+00:00) in seconds. |
| `sign` | string | Yes | Request signature generated by HMAC-SHA256. |
| `shop_cipher` | string | Yes | Cipher identifier for target shop. |

## 5. Response Structure

### Success Payload (JSON)
*   `code` (number): Status code. `0` means success.
*   `message` (string): Status description.
*   `request_id` (string): Unique log identifier.
*   `data` (object): Container.
    *   `webhooks` (array): List of active configurations.
        *   `event_type` (string): E.g., `ORDER_STATUS_CHANGE`.
        *   `address` (string): Webhook destination URL.
        *   `create_time` (number): Creation timestamp.
        *   `update_time` (number): Last update timestamp.
    *   `total_count` (number): Total registered webhooks.

## 6. Error Handling
*   Check `code` in response body.
*   If `code` is not `0`, action failed.
*   Log `request_id` for API support tickets.

## 7. Pitfalls & Best Practices
*   **Clock drift:** Sync server clock with NTP. Request fails if `timestamp` differs from server time by more than tolerance limit.
*   **Signature generation:** Sort query parameters alphabetically before hashing. Do not include headers in signature base string.
*   **Cipher usage:** Use `shop_cipher` from authorization flow. Do not use raw shop ID.
