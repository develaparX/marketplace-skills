# Send Message API Implementation Guide

Send message to buyer in conversation. Use for automated support, order updates, coupon delivery.

## Endpoint

*   **Method**: `POST`
*   **URL**: `https://open-api.tiktokglobalshop.com/customer_service/202606/conversations/{conversation_id}/messages`

## Headers

*   `content-type`: `application/json`
*   `x-tts-access-token`: Seller access token (from Get Access Token, where `user_type = 0`)

## Parameters

### Path Parameters
*   `conversation_id` (string, required): Target conversation ID.

### Query Parameters
*   `app_key` (string, required): Unique app key.
*   `sign` (string, required): Signature generated by algorithm.
*   `timestamp` (number, required): Unix timestamp GMT (UTC+00:00).
*   `shop_cipher` (string, required): Shop identifier.

### Request Body
*   `type` (string, required): Message type. Allowed: `TEXT`, `IMAGE`, `VIDEO`, `PRODUCT_CARD`, `ORDER_CARD`, `RETURN_REFUND_CARD`, `COUPON_CARD`, `LOGISTICS_CARD`.
*   `content` (string, required): Message content. **Must be JSON serialized string**.
*   `sender_role` (string, optional): Sender identity. Allowed: `CUSTOMER_SERVICE`, `SYSTEM`, `ROBOT`.

## Error Handling

| Code | Description | Action |
| :--- | :--- | :--- |
| `45101002` | Invalid params (RETURN_REFUND_CARD) | Check card payload format |
| `45101004` | Daily quota reached (10000/day) | Stop requests until next day |
| `45101006` | Sensitive content blocked | Filter message text before sending |
| `45102007` | No conversation permission | Refresh token, check shop cipher |
| `45111007` | Buyer coupon limit reached | Do not retry coupon send |
| `45111012` | Coupon QPS limit exceeded (0.5 QPS) | Implement rate limiter, retry |

---

## Pitfalls & Best Practices

*   **Double Serialization**: `content` field requires escaped JSON string.
    *   *Wrong*: `"content": { "content": "hello" }`
    *   *Right*: `"content": "{\"content\": \"hello\"}"`
*   **Rate Limits**: Coupon cards restricted to 0.5 QPS. Use queue to throttle coupon messages.
*   **Token Expiry**: Handle expired `x-tts-access-token` by refreshing token when API returns auth errors.
