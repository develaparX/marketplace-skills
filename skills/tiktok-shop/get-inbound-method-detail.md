# Get Inbound Method Detail Implementation Guide

## 1. Purpose
API gets placement details for inbound plan. Use after calling `ListAvailableInboundMethod` to retrieve specific plan option details.

## 2. Endpoint
* **Method:** `POST`
* **URL:** `https://open-api.tiktokglobalshop.com/fbt/202602/inbound_method_detail`

## 3. Headers & Auth
* `content-type`: `application/json`
* `x-tts-access-token`: Seller access token (from Get Access Token API, when `user_type` = 0).

## 5. Response Structure
* `code` (number): Success or failure status code.
* `message` (string): Status message.
* `request_id` (string): Request log identifier.
* `data` (object): Return payload.
  * `info` (array): Shipment allocation list.
    * `allocation_shipments` (array): Shipments details.
      * `warehouse_id` (string): Destination warehouse identifier.
      * `goods_items` (array): Items allocated.
        * `goods_id` (string): Item identifier.
        * `quantity` (number): Item quantity.
      * `cartons` (array): Carton details.
        * `carton_type` (string): Carton type (e.g., `SINGLE_SKU`).
        * `items` (array): Carton contents.
          * `goods_id` (string): Item identifier.
          * `quantity` (number): Item quantity.
          * `lot_code` (string): Lot code.

## 6. Error Codes
* `36009003`: Internal error. Retry request. Contact support if error persists.
* `39001002`: Empty parameters. Verify request body and query parameters.

## 7. Pitfalls & Best Practices
* **Time Window Match:** `estimate_arrival_time` values must match `available_time_window_list` from `ListAvailableInboundMethod` API response exactly. Incorrect window causes validation failure.
* **Signature Generation:** Generate `sign` query parameter using correct algorithm before sending request.
