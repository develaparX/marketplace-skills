# API Guide: Update Shipping Info

## 1. Function & Usage
API updates tracking number and carrier for shipped order. Use when carrier changes or typo correction needed.

## 2. Endpoint
*   **Method:** `POST`
*   **Path:** `/fulfillment/202309/orders/{order_id}/shipping_info/update`

## 3. Headers & Auth

*   `x-tts-access-token`: Seller access token (from Get Access Token, user_type = 0).

## 6. Error Handling
Handle specific error codes:
*   `21011004`: Not seller-shipped package. API blocks platform-shipped orders.
*   `21011007`: Active after-sale request. Process after-sale first.
*   `21011022`: Invalid tracking number or provider ID. Check inputs.
*   `21011030`: Package already delivered. Contact platform support.
*   `21011043`: Package not shipped yet. Ship package first.
*   `21011041` / `21011042` / `25001010`: Order cancelled or returned. Update blocked.

## 7. Pitfalls & Best Practices
*   **Order State:** Verify order status is "Shipped" but not "Delivered" before call.
*   **Carrier Validation:** Verify `shipping_provider_id` exists in carrier list. Invalid ID triggers error `21011022`.
*   **After-Sale Blocks:** Check for active returns. Resolve returns before update.
