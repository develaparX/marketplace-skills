# API Integration Guide: Get Shoppable Video Status

## 1. Purpose
Check status of shoppable video post. Use after calling [Publish Shoppable Video] to verify success or failure.

## 2. Endpoint
*   **Method**: `GET`
*   **Base URL**: `https://open-api.tiktokglobalshop.com`
*   **Path**: `/affiliate_creator/202509/videos/{video_id}/status`

## 3. Headers & Auth
*   `content-type`: `application/json`
*   `x-tts-access-token`: Creator access token (`user_type = 1`). Get from [Get Access Token API](https://partner.tiktokshop.com/docv2/page/678e3a3292b0f40314a92d75#Get Access Token API).

### Query Parameters

## 5. Response Structure
*   `code` (number): Success or failure code. `0` means success.
*   `message` (string): Status message. Contains failure details.
*   `request_id` (string): Request log identifier.
*   `data` (object):
    *   `video` (object):
        *   `id` (string): Video ID.
        *   `post_status` (string): Post status (e.g., `FAIL`).
        *   `post_time` (number): Unix timestamp of post.

## 6. Error Codes
*   `38007001`: System Error. Retry request.
*   `170001016`: Video does not belong to creator or video is not photo post. Check video status.

## 7. Pitfalls & Best Practices
*   **Token Type**: Must use creator token (`user_type = 1`). Merchant token causes auth failure.
*   **Signature**: Generate signature using exact query parameters and path. Incorrect order causes signature mismatch.
*   **Polling**: Do not spam API. Implement exponential backoff when checking status.
