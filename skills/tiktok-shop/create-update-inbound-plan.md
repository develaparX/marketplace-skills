# Implement Create/Update Inbound Plan API

API create or update inbound plan. Define goods, carton configurations, dimensions. Use when send inventory to FBT warehouse.

## Endpoint

*   **Method**: `POST`
*   **URL**: `https://open-api.tiktokglobalshop.com/fbt/202602/create_update_inbound_plan`

## Headers

*   `content-type`: `application/json`
*   `x-tts-access-token`: Seller access token (from Get Access Token, when `user_type` = 0).

## Parameters

### Query Parameters

| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique key assigned to app. |
| `sign` | string | Yes | Signature generated by algorithm. |
| `timestamp` | number | Yes | Unix timestamp GMT (UTC+00:00). |
| `shop_cipher` | string | Yes | Shop identifier. |

### Request Body (JSON)

| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `plan_id` | string | No | Omit to create plan. Provide to update plan. |
| `idempotent_key` | string | No | Prevent duplicate requests. |
| `goods_items` | array | No | Goods item list. |
| `cartons` | array | No | Carton list. |

## Response Structure

*   `code` (number): Status code. `0` means success.
*   `message` (string): Status message.
*   `request_id` (string): Log identifier.
*   `data` (object): Contains `inbound_plan` object with `plan_id`.

## Error Codes

*   `36009003`: Internal error. Retry request. Contact support if fail.
*   `39001002`: Empty request or parameters. Check input.

## Pitfalls & Best Practices

*   **Creation vs Update**: Omit `plan_id` to create. Include `plan_id` to update. Wrong state cause error.
*   **Idempotency**: Use `idempotent_key` to avoid double creation on network retry.
*   **Time Sync**: Ensure server clock syncs with UTC. Bad `timestamp` cause signature failure.
