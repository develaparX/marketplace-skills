# Confirm Inbound Method API Implementation Guide

## 1. Purpose & Usage
API finalizes warehouse placement option for Fulfilled by TikTok (FBT) inbound plan. Use after selecting shipment splits and destinations to lock plan details.

## 2. Endpoint
* **HTTP Method:** `POST`
* **Path:** `/fbt/202603/confirm_inbound_method`
* **Base URL:** `https://open-api.tiktokglobalshop.com`

## 3. Headers & Auth
* `content-type`: `application/json`
* `x-tts-access-token`: Seller access token (from Get Access Token, `user_type = 0`).

## 4. Parameters

### Query Parameters
| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique app key. |
| `sign` | string | Yes | Signature generated by algorithm. |
| `timestamp` | number | Yes | Unix timestamp GMT (UTC+00:00). |
| `shop_cipher` | string | Yes | Shop cipher for target shop. |

### Body Parameters
| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `plan_id` | string | Yes | Plan identifier. |
| `placement_option_id` | string | Yes | Placement option ID. |
| `inbound_method` | string | Yes | Values: `D2FC` (Direct to fulfillment center) or `ONE_HUB` (Single destination). |
| `estimate_arrival_time` | object | Yes | Contains `start_timestamp` and `end_timestamp`. |

## 5. Response Structure
* `code` (number): Status code. `0` means success.
* `message` (string): Status message.
* `request_id` (string): Request log identifier.
* `data` (object): Return payload.
  * `data.inbound_order_ids` (array of strings): Generated inbound order identifiers.

## 6. Error Handling
* `36009003`: Internal error. Retry request.
* `39001002`: Empty request or parameters. Check input payload.

## 7. Pitfalls & Best Practices

> [!WARNING]
> **CRITICAL: Confirming the inbound method is an irreversible action. Once confirmed, the placement option, shipment splits, and destinations are locked. You cannot modify these details. Ensure all parameters are correct before executing.**

* **Time Window Match:** `estimate_arrival_time` values must match `available_time_window_list` from `ListAvailableInboundMethod` API response exactly. Mismatched timestamps cause API failure.
