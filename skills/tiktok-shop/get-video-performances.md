# TikTok Video Performances API Implementation Guide

## 1. Overview
API fetch TikTok E-commerce video metrics. US creators only. Use to track GMV, orders, CTR, items sold.

## 2. Endpoint
* **Method:** `GET`
* **URL:** `https://open-api.tiktokglobalshop.com/analytics/202403/videos/performances`

## 3. Headers & Auth
* `content-type`: `application/json`
* `x-tts-access-token`: Creator access token (from Get Access Token, where `user_type = 1`)

## 4. Query Parameters

| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique app key. |
| `sign` | string | Yes | Signature generated by algorithm. |
| `timestamp` | number | Yes | Unix timestamp GMT (UTC+00:00). |
| `video_ids` | string | Yes | Comma-separated video IDs. Max 100. |
| `start_time_ge` | number | Yes | Start Unix timestamp. Max 180 days ago. Must have 1-day delay from today. |
| `end_time_le` | number | Yes | End Unix timestamp. Must have 1-day delay from today. |

## 6. Error Handling
Check `code` field in response. 
* `code == 0`: Success.
* `code != 0`: Failure. Read `message` for error details. Log `request_id` for debugging.

## 7. Pitfalls & Best Practices
* **US Only:** API works for US creators only. Request fails for other regions.
* **Data Latency:** Data has 1-day delay. Set `start_time_ge` and `end_time_le` at least 24 hours in past.
* **Query Limit:** Max 100 video IDs per request. Split larger lists into multiple batches.
* **History Limit:** Data limited to past 180 days. Querying older dates returns error.
