# Mark Conversation Read API Implementation Guide

## 5. Response Structure
*   `code` (number): Status code. `0` means success.
*   `message` (string): Status message.
*   `request_id` (string): Request log ID.
*   `data` (object): Return payload.
    *   `failed_conversation_ids` (array): IDs failed to mark read.

## 6. Error Codes
*   `16032001`: Invalid parameter Creator and Seller. Region mismatch. Check account regions.
*   `36009003`: Internal error. Retry request.

## 7. Pitfalls & Best Practices
*   **Path Typo:** Path uses `/conversatons/` (missing 'i'). Use exact path.
*   **Batch Limit:** Max 20 IDs. Split larger lists into multiple requests.
*   **Partial Success:** API returns code `0` even if some IDs fail. Check `failed_conversation_ids` in response.
