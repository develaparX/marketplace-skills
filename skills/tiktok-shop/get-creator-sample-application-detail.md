# Creator Sample Application Detail API Implementation Guide

## 1. Overview
API gets detail of creator sample application. Use to check application status, fulfillment state, order association.

## 2. Endpoint
*   **Method:** `POST`
*   **Path:** `https://open-api.tiktokglobalshop.com/affiliate_creator/202412/sample_applications/single_query`

## 3. Headers & Auth
*   `content-type`: `application/json`
*   `x-tts-access-token`: Creator access token (from Get Access Token, where `user_type = 1`).

## 4. Parameters

### Query Parameters
| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique developer application key. |
| `sign` | string | Yes | Signature generated by algorithm. |
| `timestamp` | number | Yes | Unix timestamp GMT (UTC+00:00) in seconds. |

### Body Parameters (JSON)
| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `product_id` | string | Yes | Product identifier. |
| `application_type` | string | Yes | Type of sample application. Values: `FREE_SAMPLE`, `SAMPLE_COUPON`, `SAMPLE_CAMPAIGN`. |
| `application_id` | string | Conditional | Required if `application_type` is `FREE_SAMPLE`. |
| `main_order_id` | string | Conditional | Required if `application_type` is `SAMPLE_COUPON`, `SAMPLE_CAMPAIGN`, or `REFUNDABLE_SAMPLE`. |

## 5. Response Structure
Returns standard envelope with `code`, `message`, `request_id`, and `data`.

### Key Data Fields (`data.sample_application`)
*   `id`: Application identifier.
*   `status`: Application status (e.g., `PENDING`).
*   `sample_product`: Object containing product `id`, `sku_id`, and `sku_sale_property_value_names`.
*   `creator_fulfillment`: Object containing fulfillment `status`, `expiration_time`, and `total_suspend_duration`.

## 6. Error Handling
*   **Code `36009003`**: Internal error. Action: Retry request. If fail, contact support.
*   **Non-zero Code**: Check `message` field for failure reason.

## 7. Pitfalls & Best Practices
*   **Conditional Parameters**: Match `application_id` or `main_order_id` to correct `application_type`. Wrong pairing causes validation failure.
*   **Token Type**: Use creator token (`user_type = 1`). Seller tokens fail auth.
*   **Timestamp Drift**: Ensure system clock syncs with UTC. Large drift causes signature failure.
