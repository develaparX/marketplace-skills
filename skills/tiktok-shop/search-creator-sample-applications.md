# Search Creator Sample Applications API

API searches creator sample applications. Use to track sample requests, check shipping status, manage creator fulfillment.

## Endpoint

*   **Method:** `POST`
*   **URL:** `https://open-api.tiktokglobalshop.com/affiliate_creator/202412/sample_applications/search`

## Headers & Authentication

*   `content-type`: `application/json` (Required)
*   `x-tts-access-token`: Creator access token. Get from token API where `user_type = 1`. (Required)

## Parameters

### Query Parameters

| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique application key. |
| `sign` | string | Yes | Signature generated by algorithm. |
| `timestamp` | number | Yes | Unix timestamp GMT (UTC+00:00). |
| `page_token` | string | No | Pagination offset. Leave empty for first request. |
| `page_size` | number | No | Page size limit. Range: 1-50. Default: 20. |

### Body Parameters

| Name | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `application_statuses` | array[string] | No | Filter by application status. Allowed values below. |

**Allowed Status Values:**
`PENDING`, `AWAITING_SHIPMENT`, `SHIPPED`, `CONTENT_PENDING`, `REJECT_CANCELLED`, `OVERDUE_CANCELLED`, `UNFULFILL_CANCELLED`, `DEL_OPEN_COLLAB`, `SELLER_NOT_SHIP_CANCELLED`, `WITHDRAW_CANCELLED`, `UNFULFILLABLE_CANCELLED`, `OPS_CANCELLED`, `OPS_FAILED`, `OPS_COMPLETED`, `COMPLETED`.

## Response Structure

*   `code` (number): Status code. `0` means success.
*   `message` (string): Error details or success message.
*   `request_id` (string): Request log identifier.
*   `data` (object):
    *   `next_page_token` (string): Token for next page. Empty if no more pages.
    *   `sample_applications` (array):
        *   `id` (string): Application ID.
        *   `sample_product` (object): Product details (ID, SKU ID, SKU property names).
        *   `main_order_id` (string): Associated order ID.
        *   `activity_id` (string): Campaign/activity ID.
        *   `status` (string): Application status.
        *   `creator_fulfillment` (object): Fulfillment details (ID, expiration time, suspend duration, status, bound product status).

## Error Handling

*   Check `code` field. If not `0`, request failed.
*   Read `message` for failure reason.
*   Save `request_id` for support logs.

## Pitfalls & Best Practices

*   **Page Size Limit:** Do not exceed `50` for `page_size`. Request fails if value too large.
*   **Pagination:** First request must omit `page_token`. Use `next_page_token` from response for subsequent pages.
*   **Signature:** Generate `sign` using exact query parameters. Wrong signature causes auth failure.
