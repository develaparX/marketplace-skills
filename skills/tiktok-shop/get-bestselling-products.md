# Bestselling Products API Implementation Guide

### 1. Purpose
API returns top 100 performing products for target date range. Use for market analysis, inventory planning, trend tracking.

### 2. Endpoint
*   **Method**: `GET`
*   **Base URL**: `https://open-api.tiktokglobalshop.com`
*   **Path**: `/analytics/202511/products/bestselling`

### 3. Headers & Authentication
*   `content-type`: `application/json`
*   `x-tts-access-token`: Seller access token (requires `user_type = 0`)

### 4. Query Parameters
| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `app_key` | string | Yes | Unique application key. |
| `sign` | string | Yes | Signature generated by signature algorithm. |
| `timestamp` | number | Yes | Unix timestamp GMT (UTC+00:00) in seconds. |
| `shop_cipher` | string | Yes | Shop identifier cipher. |
| `date` | string | Yes | Reference date (ISO 8601 `YYYY-MM-DD`) in shop timezone. |
| `time_slot` | string | Yes | Query time range. Allowed: `1D`, `7D`, `30D`. |
| `category_id` | string | No | Filter by L1 category ID only. |
| `currency` | string | No | `USD` or `LOCAL`. |

### 5. Response Structure
Returns JSON object:
*   `code` (number): Success indicator. `0` means success.
*   `message` (string): Status description.
*   `request_id` (string): Unique request identifier for logs.
*   `data` (object): Payload.
    *   `products` (array): Top products.
        *   `rank` (number): Performance rank (1-100).
        *   `id` (string): Product ID.
        *   `name` (string): Product title.
        *   `gmv_range` (string): Estimated GMV range.
        *   `rating` (string): Product rating.
        *   `shop_name` (string): Seller shop name.
        *   `shop_id` (string): Seller shop ID.
        *   `product_image` (object): Image metadata (height, width, URI, URLs).

### 6. Error Handling
*   Check `code` field in response.
*   If `code` is not `0`, request failed.
*   Log `request_id` and `message` for troubleshooting.

### 7. Pitfalls & Best Practices
*   **Category Filter**: `category_id` accepts L1 (Level 1) categories only. Subcategories cause empty results or errors.
*   **Timezone Alignment**: `date` parameter must match shop registered timezone. Incorrect offset shifts data window.
*   **Signature Generation**: Sort all query parameters alphabetically before generating `sign` hash.
*   **Token Expiry**: Refresh `x-tts-access-token` before expiry to prevent `401` unauthorized errors.
