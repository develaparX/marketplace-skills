# Download Compass Task File API Guide

## 1. Purpose & Use Case
Download result file of completed export task. Use after task finishes. Requires ownership validation.

## 2. Endpoint & Method
*   **Method:** `GET`
*   **Base URL:** `https://open-api.tiktokglobalshop.com`
*   **Path:** `/affiliate_seller/202603/compass/offline_tasks/{task_id}/file`

## 3. Headers & Auth
*   `content-type`: `multipart/form-data` (Required)
*   `x-tts-access-token`: Seller access token. Get via seller authorization (user_type = 0).

## 4. Parameters

### Path Parameter
*   `task_id` (string, required): ID of export task.

### Query Parameters
*   `app_key` (string, required): Unique app identifier.
*   `sign` (string, required): Signature generated by algorithm.
*   `timestamp` (number, required): Unix timestamp in seconds (UTC).
*   `shop_cipher` (string, required): Shop identifier. Get via Get Authorization Shop API.

## 5. Response Structure
Returns JSON object.

*   `code` (number): Status code. `0` means success.
*   `message` (string): Status or error message.
*   `request_id` (string): Request log identifier.
*   `data` (object): Payload.
    *   `file` (object): File container.
        *   `base64` (string): Base64 encoded file data.

### Example Response
```json
{
  "code": 0,
  "data": {
    "file": {
      "base64": "ssssssssssssssaaaaaaaaaaa"
    }
  },
  "message": "Success",
```

## 6. Error Handling
Check `code` value. If non-zero, action failed. Read `message` for failure reason. Log `request_id` for debugging.

## 7. Pitfalls & Best Practices
*   **Shop Cipher:** Wrong cipher causes empty or incorrect data for cross-border shops. Always fetch fresh cipher.
*   **Memory Limit:** Base64 string can be large. Decode stream directly to file to prevent memory exhaustion.
*   **Signature:** Include path parameters in signature calculation.
