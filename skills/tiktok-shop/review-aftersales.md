# Review Aftersales API Implementation Guide

## 1. Overview
API reviews buyer return requests. Use to approve, reject, or offer alternatives (e.g., partial refund).

## 2. Endpoint & HTTP Method
*   **Method:** `POST`
*   **Path:** `/return_refund/202603/aftersales/review`
*   **Base URL:** `https://open-api.tiktokglobalshop.com`

## 3. Headers & Authentication
### Headers
*   `content-type`: `application/json`
*   `x-tts-access-token`: Seller access token (when `user_type` = 0)

### Query Parameters (Auth)
*   `app_key`: Unique app key.
*   `sign`: Signature generated by algorithm.
*   `timestamp`: Unix timestamp GMT (UTC+00:00).
*   `shop_cipher`: Shop identifier.

## 4. Request Parameters

### Body Parameters (JSON)
*   `aftersales_request_decisions` (array, optional): Decisions to apply.
    *   `request_id` (string, required): Aftersales request ID.
    *   `request_id_type` (string, required): ID type. Example: `AFTERSALES`.
    *   `line_item_decisions` (array, required): Decisions per line item.
        *   `line_items` (array, required): Target items.
            *   `return_line_item_id` (string, required): Return line item ID.
            *   `sub_return_line_item_id` (string, required): Sub return line item ID.
        *   `action` (string, required): Action to take. Example: `APPROVE_REQUEST`.
        *   `alternative_solution_details` (object, optional): Details for alternative solutions.
            *   `partial_refund_amount` (string, optional): Refund amount. Example: `12.34`.
            *   `currency` (string, optional): Currency code. Example: `USD`.

## 5. Response Structure
*   `code` (number): Status code. `0` means success.
*   `message` (string): Success or failure message.
*   `request_id` (string): Request log ID.
*   `data` (object): Specific return info.
    *   `errors` (array, optional): List of processing errors.
        *   `code` (string): Error code.
        *   `message` (string): Error description.

## 6. Error Codes
*   `11050001`: Invalid param.
*   `25001001`: Invalid request parameter type/format.
*   `25001003`: Invalid order status.
*   `25001028`: Reverse lock failed.
*   `25001044`: ConfirmReceive fulfillment order failed.
*   `25001058`: Reverse downstream data load error.
*   `25007006`: Order not found.
*   `25007007`: ReverseOrderExistNegotiate.
*   `25011010`: Request transferred to platform operator.
*   `25011012`: Reverse status cannot approve application.
*   `25011013`: Reverse status cannot approve receive.
*   `25011023`: Forbid process authentic guarantee order.
*   `25011026`: Forbid process receive for FBT order.
*   `25011032`: ReverseInvalidStatusPartialRefund.
*   `25020004`: Decision not match reverse type.
*   `25020005`: No permission.
*   `25020008`: Internal error.
*   `25020009`: Reverse already cancelled.

## 7. Pitfalls & Best Practices
*   **FBT Orders:** Cannot process receive for FBT (Fulfilled by TikTok) orders. Will trigger error `25011026`.
*   **Status Check:** Verify order status before call. Avoid `25001003` or `25011012`.
*   **Partial Success:** Response `code` can be `0` but `data.errors` contains item errors. Check both.
*   **Lock Errors:** `25001028` means concurrent updates. Retry request.
